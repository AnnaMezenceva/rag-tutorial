# Workflow

## Цикл одной итерации
1. Прочитать задание итерации
2. Написать / изменить код
3. Проверить командой из задания
4. Закоммитить результат
5. Обновить статус в tasklist.md
6. Перейти к следующей итерации

## Что делать до кода
- Прочитать задание и понять ожидаемый результат
- Определить, какие файлы нужно изменить

## Правила коммита
- Один коммит на итерацию
- Понятное сообщение: "iter 03: chunking"

## Запрещённые действия
- Не пропускать итерации
- Не менять файлы, не относящиеся к текущей итерации
