# Как улучшить MVP

## Выбранные улучшения

### 1) Embeddings вместо TF-IDF
- **Зачем:** TF-IDF ищет по совпадению слов, а не по смыслу. Запрос «кино о любви» не найдёт фильм с описанием «романтическая история». Sentence-transformers решают эту проблему.
- **Какие файлы менять:** `build_index.py`, `retriever.py`
- **Как проверить:** сравнить результаты поиска на одних и тех же вопросах — семантический поиск должен находить релевантные фильмы даже без точного совпадения слов.

### 2) Подключение LLM
- **Зачем:** сейчас ответ — просто список найденных фрагментов. С LLM система будет формулировать связный ответ на естественном языке.
- **Какие файлы менять:** `app/generator.py`, `app/prompts.py`
- **Как проверить:** ответ должен быть читаемым текстом, а не набором чанков, при этом основанным только на найденном контексте.

### 3) Улучшение интерфейса Streamlit
- **Зачем:** сейчас UI минимальный — поле ввода и список фрагментов. Фильтры по жанру, порог score, история запросов и подсветка совпавших слов сделают систему удобнее для пользователя.
- **Какие файлы менять:** `app/main.py`
- **Как проверить:** пользователь может фильтровать результаты, видеть историю запросов и быстро оценивать релевантность по подсветке.

### 4) Оценка качества (Eval)
- **Зачем:** ручная проверка demo-вопросами не масштабируется. Автоматические метрики (retrieval@k, hit-rate) позволяют объективно сравнивать изменения в pipeline.
- **Какие файлы менять:** новый скрипт `scripts/eval.py`, набор эталонных вопросов с ожидаемыми doc_id
- **Как проверить:** запуск скрипта выдаёт числовые метрики, которые можно сравнить до и после изменений.
